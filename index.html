Patterns - разгрузка классов.

Шаблоны, которые применял в коде

QueryObject (запросы в одном месте)

class FindTeams
    attr_accessor :initial_scope

    def initialize(initial_scope)
        @initial_scope = initial_scope
    end

    def call(params)
        scoped = search_by_name(initial_scope, params[:name])
        ...
        scoped
    end

    private

    def search_by_name(scoped, name = nil)
        return scoped if name.blank?

        scoped.where("name ILIKE '%?%'", name)
    end

    ...
end

ServiceObject (для бизнес логики)

class ApplicationService
    def self.call(*args, &block)
        new(*args, &block).call
    end
end

Interactor (комбинирование процессов бизнес логики)

class MakeOffer
    include Interactor::Organizer

    organize CreateOrder,
             CalculateOffer,
             SendOffer
end

СУБД postgresql

Есть:

  -понимание индексов, которые нужны для ускорения поиска

  -практика составления комплексных SQL запросов с применением подзапросов, CTE, JOIN, UNION, HAVING и многого другого на sql-ex.ru

  -понимание, что оптимизация не всегда может лежать на стороне запросов и не всегда нужно смотреть планировщик через Explain analyze. При хранении в БД файлов или длинных текстовых полей с использованием сжатия по умолчанию, может заполнится нормальный объём данных, который будет нагружать CPU (статья, где человек снизил нагрузку, убрав запись в TOAST)

Или если вместо Pool соединений, мы регулярно открываем и закрываем одиночные соединения, то на каждое такое соединение PostgreSQL будет выделять кэш и потом нужно будет это очищать, что делает нагрузку на СУБД (отсылка на идею в книге - PostgreSQL 15 Изнутри)
